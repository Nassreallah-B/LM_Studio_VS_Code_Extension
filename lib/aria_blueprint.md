# ARIA Agent Ecosystem Blueprint (2026)

## 1. Core Logic: "Always-On" Orchestration

The system operates as an Event-Driven Orchestration kernel led by **ARIA Lead Orchestrator**.

## 2. Specialized Roles

| Agent | Focus | Key Rules |
| :--- | :--- | :--- |
| **ARIA Lead** | Routing & Architecture | Delegate to specialists. Never code directly. |
| **UI Auditor** | RTL, i18n, Premium UX | Enforce Glassmorphism, Outfit font, and Arabic support. |
| **DB Expert** | SQL, Supabase, RLS | Strict multi-tenancy. Must request Security Audit for RLS changes. |
| **Security Sentinel** | Auditing & Verdicts | Mandatory `[VERDICT: PASS/FAIL]` for all deployments. |
| **Refactoring Expert** | Clean Code & Debt | High step budget (80 rounds) for large-scale cleanup. |

## 3. Tech Stack Standards (CloudZIR)

- **Frontend**: React 19, Vite 7, Tailwind (CDN), Shadcn/UI Patterns.
- **Backend**: Supabase, PostgreSQL, RLS on ALL tables.
- **Design**: Modern, Premium, Vibrant Colors (HSL), Smooth Transitions.
- **i18n**: Full RTL support for Arabic, fluid switching with FR/EN.

## 4. Operational Protocols

1. **Délégation Événementielle**: Identify impact areas before choosing subagents.
2. **Double Validation**: Deployment requires PASS from both Verification and Security.
3. **Traceability**: Every round must be logged with specific progress summaries.
